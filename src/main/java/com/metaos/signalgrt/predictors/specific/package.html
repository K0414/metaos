Base predictors for specific usages.

If you are a user, please go directly to subpackages containing concrete implementations to predict specific elements.

If you're developer, you should look for a predictor that performs a similar task in this package and extend it fixing initialization parameters.

The main ideas of classes in this package are:

  • Classes OnePredictorPerTypeOfDayOfWeek, OnePredictorPerBin and OnePredictorPerBinAndTypeOfDayOfWeek are wrappers of several predictors of the same type given at construction time.
  • Classes PCACombiningSeveralDays and PCADayByDayPredictor act as base classes for other Principal Component Analysis: the first one takes data creating a matrix with one day per column (recording information for several days and one one product) and the second one takes data to create a matrix with one product daily data per column (recording information for several products in only one day).
  • StaticDayByDayPredictor is the base class for predictors like PCADayByDayPredictor that after recording all information for a day, generate a prediction for the next day as a whole.
  • In the opposite, DynamicIntradePredictorListener is the base class of predictors that generate predictions while they are learning.
  • QuasiDynamicIntradayPredictorListener uses some data learnt at the begining of the day to forecast the whole day.
